Projects

Hacker News Live Comments Bookmarklet

Project Home Page - https://hnlivecomments.pex2.jp/
Github - https://github.com/harmony7/HNLiveComments

Hacker News is a great source of information but one of its problems is its static nature. In this project I have written a bookmarklet that lets you see newly posted comments on Hacker News without having to refresh the page. It uses HNStream (made by Justin of Fanout.io) to obtain the live data, and it fiddles with the page DOM to display new comments in a way that very closely matches HN's look.

It is a JavaScript bookmarklet that does some HTML scraping to turn the view into a KnockoutJS view model. It also uses the Pollymer Ajax library to perform long polling and request retries.

docpad-plugin-vash

NPM - https://npmjs.org/package/docpad-plugin-vash
Github - https://github.com/harmony7/docpad-plugin-vash

I wrote this plugin for DocPad as I began to work on this blog. It is a plugin that transforms Vash to any file type.

Vash is an implementation of the Razor syntax view engine for JavaScript. This plugin allows you to use the Razor syntax to write your DocPad layouts.

WebhookInbox

Project Home Page - http://webhookinbox.com/
Github - https://github.com/fanout/webhookinbox

I implemented much of the frontend components of WebhookInbox, a web service and website that receives HTTP requests from external sources and stores the data of those requests for later retrieval and examination.

WebhookInbox makes it possible to create a temporary HTTP endpoint, which can be passed to other applications for use during development. If you're developing a service that makes HTTP callbacks, WebhookInbox can be used as a convenient way to examine what you're sending. Incoming request data is displayed on the WebhookInbox website in realtime as it happens, thanks to the Pollymer Library. The data is orchestrated on in the browser using the AngularJS Framework.

There is an instance of WebhookInbox running at http://webhookinbox.com/ for anyone to use.

nodegrip

NPM - https://npmjs.org/package/grip
Github - https://github.com/fanout/nodegrip

I authored the nodegrip library, which is an interface library for Node.js for supporting the Generic Realtime Intermediary Protocol (GRIP). GRIP is a technique for communicating with an HTTP proxy server to hold open an incoming connection.

This library is designed to be used with HTTP reverse proxy servers that support the GRIP interface, such as Pushpin, from Fanout.io.

nodefpp

NPM - https://npmjs.org/package/fpp
Github - https://github.com/fanout/nodefpp

I authored the nodefpp library, which is a library for Node.js for supporting the Fanout Pubsub Protocol (FPP). The Fanout pubsub protocol is used to send simple realtime messages over a communications channel. This library has been designed for use either with Fanout's pubsub service or any compatible service.

This library implements the FPP data format as a custom data format for nodepubcontrol, which is an EPCP library for Node.js and is another of my projects (listed below).

nodepubcontrol

NPM - https://npmjs.org/package/pubcontrol
Github - https://github.com/fanout/nodepubcontrol

I authored the pubcontrol library, which is a library for Node.js for supporting message publishing using the HTTP Extensible Pubsub Control Protocol (EPCP), which defines a generalized and extensible data publishing protocol using HTTP.

This library contains a Format base class and a PubControl class that is used to send messages. This library also supports the case when the EPCP endpoint requires certain types of authentication.

Pollymer

Github - https://github.com/fanout/pollymer

I co-authored Pollymer, a general-purpose JavaScript library that provides conveniences for performing Ajax operations involving long-polling applications, such as request retries, exponential backoff between requests, randomized request delaying, and workarounds for browser "busy" indications. It also implements multiple transports to ensure cross-domain access works in all major browsers.

WebSockHop

Github - https://github.com/fanout/websockhop

I authored WebSockHop, a small convenience library for WebSocket clients that provides automatic reconnect, periodic pinging, and request/response interactions.
